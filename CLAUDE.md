# CLAUDE.md — سياق مشروع «دليل آية الغذائي»

> هذا الملف يقرأه Claude Code تلقائيًا. لا تحذفه.

## نبذة
تطبيق ويب شخصي (PWA) لمتابعة تغذية الحمل، أعدّه أبو مانع (أحمد) لزوجته **آية** (حامل في الشهر الثاني ~٨ أسابيع). الهدف: دليل دافئ + قائمة مهام يومية + عدّاد ماء + تذكيرات، مع مزامنة سحابية تتيح لأبو مانع متابعة تقدّمها من جواله.

## صاحب المشروع وتفضيلاته
- خاطبه بالعربية الخليجية، بأسلوب **مرقّم ومباشر**.
- يحب الردود المنظّمة والخطوات العملية الواضحة.
- لا تنشر/ترفع أي تغيير قبل موافقته الصريحة.

## التقنية (مهم: لا تعقّدها)
- **HTML/CSS/JS صرف** في ملف واحد لكل صفحة. **لا framework، لا build step.**
- `index.html` — تطبيق آية. `dashboard.html` — لوحة متابعة أبو مانع.
- Supabase عبر CDN (`@supabase/supabase-js@2`). المفاتيح في كتلة CONFIG أعلى كل ملف (anon key عامة بطبيعتها، محميّة بـ RLS).
- PWA: `manifest.json` + `sw.js` + أيقونتان.
- النشر: GitHub → Vercel (نشر تلقائي عند كل push).

## كتلة الإعداد (نفس القيم في الملفين)
```js
var SUPA_URL='';    // Project URL
var SUPA_KEY='';    // anon public key
var FAMILY_ID='';   // id من جدول family
```
إذا كانت فارغة، التطبيق يعمل بالحفظ المحلي فقط (localStorage) بدون أخطاء.

## قاعدة البيانات
- جدولان أساسيان: `family` و `daily_logs` (صف واحد لكل يوم، upsert على `family_id,log_date`).
- جداول الإشعارات: `reminders` (تذكيرات قابلة للتعديل من لوحة أبو مانع) و `push_subscriptions` (اشتراك جهاز آية).
- المخططات: `SUPABASE_SCHEMA.sql` (الأساس) و `PUSH_SCHEMA.sql` (الإشعارات).
- `checklist` يُخزَّن jsonb بمفاتيح: folic, wake, bfast, snk1, lunch, snk2, dinner, milk.

## الإشعارات (Push)
- جوال آية: Samsung Galaxy A25 5G (أندرويد) — Web Push مدعوم كامل.
- الإرسال عبر Supabase Edge Function `send-reminders` + Cron كل ٥ دقائق (توقيت السعودية UTC+3).
- الأوقات تُقرأ من جدول `reminders` ويتحكم بها أبو مانع من `dashboard.html`.
- تفاصيل البناء الكاملة في `PUSH_TASK.md`. `sw.js` فيه معالج `push` جاهز.
